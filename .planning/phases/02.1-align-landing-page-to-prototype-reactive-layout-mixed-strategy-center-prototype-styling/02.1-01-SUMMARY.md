## What Was Built

### Mock Data Module (src/lib/mock-data.ts)
- `MockNewsItem` interface with all required fields (id, headlineKey, category, source, date, impactScore, summaryKey, shortTermImpactKey, midTermImpactKey, relatedHoldings, relatedStrategies)
- `MockStrategyData` interface (slug, dailyChangePct, topHoldings array)
- `mockNewsItems` array: 4 items covering ai (Reuters, score 9), geopolitics (Bloomberg, score 8), macro (Financial Times, score 7), crypto (CoinDesk, score 6)
- `mockStrategyData` record: future-tech (+2.25%), traditional (+0.45%), commodities (-0.80%), crypto (+4.10%)

### I18n Keys
- `newsDigest` section added to both en.json and zh-HK.json: title, impactLabel, shortTermLabel, midTermLabel, relatedLabel, 6 categories, 4 news items (headline, summary, shortTermImpact, midTermImpact)
- `landing` section extended: dailyChange, topHoldings, portfolioSnapshot, totalValue, viewStrategy — bilingual

### NewsCard Component (src/components/landing/NewsCard.tsx)
- Category badge with 6 color mappings (geopolitics=rose, ai=violet, semiconductor=blue, commodities=amber, crypto=cyan, macro=emerald)
- Impact score badge color-coded by threshold (9-10=red, 7-8=amber, 5-6=yellow, 1-4=green)
- Short-term and mid-term impact rows with label badges
- Related holdings as outline badges

### StrategyCard Enhancement (src/components/landing/StrategyCard.tsx)
- Optional `dailyChangePct` with green/red coloring and +/- prefix formatting
- Optional `topHoldings` array rendered as "TICKER XX% · TICKER XX%" inline text
- All new props optional — existing usage on fund pages unchanged

## Decisions Made

1. Mock data uses i18n key references rather than hardcoded strings to keep the data layer locale-agnostic. Consumers resolve keys via `useTranslations` or `getTranslations`.
2. StrategyCard daily change and topHoldings are optional props guarded by `dailyChangePct !== undefined` — no existing callers need updating.
