- Import `NewsItem` from `@/types/news`

Export `GET` handler:
- `const { slug } = await params` (Next.js 16 Promise params)
- Read `locale` from URL search params: `const locale = (new URL(req.url).searchParams.get('locale') || 'en') as 'en' | 'zh-HK'`
- Validate strategy exists
- Fetch news: `const newsResult = await getNewsForStrategy(strategy)`
- Build strategy context string for AI: `strategy.allocations.map(a => a.ticker + ' ' + Math.round(a.weight * 100) + '%').join(', ')`
- For each article, run AI analysis in parallel: `Promise.allSettled(newsResult.data.map(article => analyzeNewsItem(article, strategyContext, locale)))`
- Merge: create `NewsItem[]` by combining each RawNewsItem with its analysis result (or null if rejected/failed)
- Return `Response.json({ articles, cachedAt: newsResult.cachedAt })`
- Wrap in try/catch: return 503 on total failure
- Per D-08: The locale parameter controls the language of AI output. Each locale is cached independently by the analyzer (cache key includes locale).
- Per I18N-03: Both en and zh-HK are supported via the locale query param.
